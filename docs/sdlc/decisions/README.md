## Process

New ADRs should:

1. Use the next sequential number (0004, 0005, etc.)
2. Follow the existing format:
   - Status, Date, Deciders/Owners, Tags
   - Context explaining the decision need
   - Decision statement
   - Consequences (positive/negative)
   - Alternatives considered
   - Implementation notes
3. Be added to the table above when created

## Status Values

- **Proposed**: Under discussion, not yet implemented
- **Accepted**: Approved and actively followed
- **Deprecated**: No longer applies, superseded by another ADR
- **Superseded**: Replaced by a newer ADR (link to replacement)
