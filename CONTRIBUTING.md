# Contributing

Thanks for your interest in Offload. This is currently a personal project, but
thoughtful contributions are welcome.

## Before You Start

- Open an issue to discuss any non-trivial change or new feature.
- Keep scope small and focused to reduce review time and risk.

## Making Changes

- Create a branch for your work.
- Follow existing code style and architecture patterns.
- Add intent headers to key files you touch.
- Update documentation when behavior or usage changes.
- Add tests for behavior changes where practical.

## Pull Requests

- Keep PRs small and clearly scoped.
- Use descriptive commit messages (Conventional Commits encouraged).
- Fill out the PR template and note testing performed.

## Documentation Changes

- Run `markdownlint` before committing documentation updates.

## Help

If you have questions, see `SUPPORT.md`.
